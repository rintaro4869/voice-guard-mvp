# Voiceguard 30日マネタイズ実行計画

作成日: 2026-05-11

## 目的

- 検索流入の伸びを崩さずに、収益化導線を整理する
- `無料Web → 比較ページ → 外部商品` の流れを作る
- 30日以内に「初収益の再現性」を確認する

## 30日KPI

- トップページ → `safety-plan` 遷移率: 8%以上
- 比較ページ → 外部商品クリック率: 12%以上
- `recommend-intercom` / `recommend-buzzer` 合計クリック: 週20以上
- 外部商品クリックのうち、ドアホン比率: 40%以上
- 検索流入クリック数: 週60以上を維持

## Week 1

- トップページの収益ファネルを整理する
- 再生UI直下の強すぎる広告導線を外す
- `safety-plan`、`recommend-intercom`、`recommend-buzzer` への内部導線を上げる
- `guide` を「準備・ホーム画面追加・トラブル対応」に寄せて、主戦場キーワードの食い合いを減らす

## Week 2

- `recommend-intercom` を `賃貸 / 工事不要 / 外出先応答 / 録画` 軸でリライト
- `recommend-buzzer` を `音量 / 誤作動 / 持ち歩きやすさ / デザイン` 軸でリライト
- `safety-plan` を「不安の種類から選ぶ」導線に寄せる
- 比較ページのCTA文言を「楽天で見る」だけでなく「候補を見る」「比較に戻る」に分ける

## Week 3

- クエリ別LPを3本追加する
- 候補:
  - `一人暮らし 男性の声`
  - `男の声 防犯`
  - `インターホン 応答 男性の声`
- それぞれトップページへの送客ではなく、検索意図に合わせて独立した回答を置く

## Week 4

- 外部商品クリックの実績を見て、売れるカテゴリを1つに寄せる
- ドアホンが強ければ `賃貸向け / 工事不要 / 置き配対応` で比較を増やす
- ブザーが強ければ `かわいい / 目立たない / 夜道向け` で比較を増やす
- iPhoneアプリ化の要件を確定する

## 実装済みの文言変更

- `index.html`
  - 再生UIの近くで商品に飛ばす構成をやめ、比較ページへの送客に変更
  - 「無料で使いながら、防犯対策を足したい方へ」を追加
- `guide.html`
  - 主戦場キーワードの説明ページから、準備・ホーム画面追加・トラブル対応ページへ方向転換
- `safety-plan.html`
  - 防犯グッズ比較ページとしての役割を明確化
- `recommend-intercom.html`
  - スマートドアホン比較の商用意図を強化
- `recommend-buzzer.html`
  - 防犯ブザー比較の商用意図を強化

## やらないこと

- 30日以内にトップページへAdSenseを入れない
- 緊急時の再生導線にインタースティシャル広告を入れない
- サブスク導入を先にやらない

## 次の判断基準

- 30日後に比較ページ経由の外部クリックが伸びるなら、アフィリエイト最適化を継続
- 伸びないなら、検索流入は「無料利用ニーズ」に偏っていると判断し、iPhoneアプリの買い切り検証を優先する
