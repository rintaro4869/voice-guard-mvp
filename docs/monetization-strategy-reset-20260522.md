# VoiceGuard マネタイズ方針見直し

更新日: 2026-05-22

## 現状認識

- 過去90日の楽天アフィリエイト実績は `22クリック / 売上0件 / 成果報酬0円`
- 現状のボトルネックは「商品が弱い」ことより、`無料利用ニーズ` と `購入ニーズ` のズレ
- トップページ訪問者の主目的は `今すぐ男性音声を使うこと` であり、商品購入ではない

## 方針転換

収益化の優先順位を以下に変更する。

1. `無料体験の満足度を守る`
2. `どの内部導線なら商用意図に変わるかを可視化する`
3. `比較ページだけで商品クリックを取りにいく`

トップページは収益ページではなく、`利用開始ページ` として扱う。

## 新ファネル

1. `トップページ`
   - 目的: 再生、継続利用、ホーム画面追加
   - 副目的: 比較ハブへの送客
2. `safety-plan`
   - 目的: 不安別の振り分け
   - ここでは原則、外部商品へ直送しない
3. `recommend-intercom` / `recommend-buzzer`
   - 目的: 比較後の商用クリック
   - ここだけでアフィリエイトクリックを取りにいく

## 追加した計測

`assets/monetization.js` で以下を送る。

- `monetization_landing`
  - どの内部CTAから遷移して着地したか
- `monetization_cta_click`
  - どのCTAが押されたか
  - `cta_kind`, `slot`, `page_role`, `entry_from`, `aff_key`, `aff_status` を記録
- 既存の `aff_click`
  - クリック位置に加えて、流入元と fallback 状態を記録

## 読み方

最初に見るべき指標は売上ではなく、以下。

1. `index -> safety-plan` の遷移数
2. `safety-plan -> recommend-*` の遷移率
3. `recommend-* -> aff_click` のクリック率
4. `aff_status=using_fallback` の比率

## 30日判断基準

### 継続条件

- `safety-plan` 経由の比較ページ流入が増える
- `recommend-*` で外部クリックが一定数出る
- 特定カテゴリにクリックが寄る

### 見切り条件

- トップの利用は伸びるが、比較ページ送客が弱い
- 比較ページ送客はあるが、外部クリックが増えない
- 外部クリックは増えても売上が立たない

この場合は、物販アフィリエイトを主軸にせず、次を優先する。

- iPhoneアプリ化
- 買い切りの軽量アプリ検証
- note / PDF / セーフティガイド等の低単価デジタル商品

## 商品選定ルール

- `大人女性` `通勤帰宅` `夜道` `玄関対応` に寄る商品を優先
- `小学生向け` が強すぎる商品はCVRが落ちやすい
- 売り切れ時は fallback で検索結果に逃がす
- 404や中間ページ異常がある候補は primary にしない
